# Discovery Report

## Purpose

The Discovery Report is a local, customer-readable artifact generated by Amber Lens.

## Key Design Decision

The report is generated locally before cloud-bound tokenization is required.

The customer may choose:

- `local-readable` mode: real names are shown locally.
- `privacy-safe` mode: names are tokenized or obfuscated.

## Rules

The report must be self-contained HTML, offline renderable, and free of tracking pixels, remote JavaScript, external assets, and telemetry.

## MVP Sections

1. Executive summary
2. Sovereignty / local-run receipt
3. Estate at a glance
4. Compute inventory
5. Database inventory
6. Storage summary
7. OS and end-of-life observations
8. Utilization summary
9. Preliminary right-sizing opportunities
10. SQL Server modernization observations
11. Oracle licensing / feature observations
12. Collection warnings
13. Follow-up questions
14. Navigator upgrade boundary

## Boundary

The Discovery Report may include preliminary observations. It should not include validated Operational Risk Score, detailed TCO, multi-cloud recommendation, or vendor bid recommendation.
